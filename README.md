# 16-Bit Virtual Machine in C

A lightweight 16-bit virtual machine written entirely in C.

The VM executes custom bytecode generated by a compiler written in Python. Python was chosen for the compiler because of its powerful string manipulation capabilities and ease of parsing assembly-like source code.

The architecture is inspired by the Intel 8086, featuring a shared code/stack memory space, general-purpose registers, status flags, and a fetch-decode-execute cycle.

---

## Features

- 16-bit address space
- Custom bytecode format
- Multiple addressing modes
- Stack support
- Conditional and unconditional jumps
- Arithmetic instructions with flag management
- Separate RAM area
- Simple register-based architecture

---

## Architecture

### Memory Layout

The VM provides a total address space of **65,536 bytes (64 KB)**.

```text
+-------------------------+ 0x0000
|                         |
|      Code Segment       |
|                         |
+-------------------------+
|                         |
|         Stack           |
|      (grows downward)   |
|                         |
+-------------------------+ 0xFFFF
```

Code and stack share the same memory space.

- Memory size: **65,536 bytes**
- Program Counter (`PC`) starts at `0x0000`
- Stack Pointer (`SP`) starts at `0xFFFF`
- Stack grows toward lower addresses

---

### RAM

The VM includes an additional dedicated RAM area:

- Size: **256 bytes**
- Address range: `0x00 - 0xFF`

This memory is intended for data storage during program execution.

---

## Registers

The VM contains six 8-bit registers.

| Register | Description |
|----------|-------------|
| R0 | General-purpose register |
| R1 | General-purpose register |
| R2 | General-purpose register |
| R3 | General-purpose register |
| LR (R4) | Loop register |
| AR (R5) | Arithmetic register |

### Arithmetic Register (AR)

The Arithmetic Register stores overflow values produced by arithmetic instructions.

For example, the `MUL` instruction stores:

- Low byte → `R0`
- High byte → `AR`

---

## Special Registers

### Program Counter (PC)

The Program Counter contains the address of the next instruction to execute.

- Size: 16 bits
- Initial value: `0x0000`

---

### Stack Pointer (SP)

The Stack Pointer contains the address of the top of the stack.

- Size: 16 bits
- Initial value: `0xFFFF`

---

## Flag Registers

The VM implements two status flags.

| Flag | Description |
|------|-------------|
| CF | Carry Flag |
| ZF | Zero Flag |

### Carry Flag (CF)

Set to `1` when:

- An addition overflows
- A subtraction requires a borrow
- A comparison indicates that the first operand is smaller than the second

Otherwise, it is set to `0`.

---

### Zero Flag (ZF)

Set to `1` when an operation produces a result equal to zero.

Otherwise, it is set to `0`.

---

## CPU Execution Cycle

The virtual CPU follows a classic fetch-decode-execute cycle.

1. Fetch the instruction pointed to by `PC`
2. Increment `PC`
3. Decode the opcode
4. Read operands
5. Execute the instruction
6. Update flags if necessary
7. Repeat until a `STOP` instruction is encountered

---

## Instruction Set

| Opcode | Mnemonic | Description |
|--------:|----------|-------------|
| `0x01` | LOAD | Load data into a register |
| `0x02` | ADD | Add values |
| `0x03` | SUB | Subtract values |
| `0x04` | JMP | Unconditional jump |
| `0x05` | MUL | Multiply values |
| `0x06` | PUSH | Push register onto the stack |
| `0x07` | POP | Pop value from the stack |
| `0x08` | JZ | Jump if zero |
| `0x09` | JNZ | Jump if not zero |
| `0x0A` | JC | Jump if carry |
| `0x0B` | JNC | Jump if no carry |
| `0x0C` | CMP | Compare two registers |
| `0x0D` | JE | Jump if equal |
| `0x0E` | JL | Jump if lower |
| `0x0F` | JG | Jump if greater |
| `0xFF` | STOP | Stop execution |

## Addressing Modes

Several instructions support four addressing modes.

The addressing mode is encoded in the two most significant bits of the operand descriptor.

```text
7 6 5 4 3 2 1 0
+---+-----------+
| M | Register  |
+---+-----------+
```

- Bits `7-6`: Addressing mode
- Bits `5-0`: Destination register

---

### Mode 0 — Immediate

Loads an immediate value.

```assembly
LOAD R0, 42
ADD R1, 10
SUB R2, 5
```

---

### Mode 1 — Register

Uses another register as the source operand.

```assembly
LOAD R0, R1
ADD R0, R2
SUB R3, R1
```

---

### Mode 2 — Direct RAM

Uses a RAM address as the source operand.

```assembly
LOAD R0, [0x20]
ADD R1, [0x10]
SUB R2, [0xFF]
```

---

### Mode 3 — Indirect RAM

Uses a register containing a RAM address.

```assembly
LOAD R0, [R1]
ADD R2, [R3]
SUB R0, [R2]
```

---

## Instruction Details

### LOAD

Loads a value into a destination register.

```assembly
LOAD destination, source
```

Updates no flags.

---

### ADD

Adds the source operand to the destination register.

```assembly
ADD destination, source
```

Updates:

- `CF`
- `ZF`

---

### SUB

Subtracts the source operand from the destination register.

```assembly
SUB destination, source
```

Updates:

- `CF`
- `ZF`

---

### MUL

Multiplies `R0` by another register.

```assembly
MUL R1
```

Equivalent to:

```text
R0 × R1
```

Result:

```text
R0 = low byte
AR = high byte
```

Flags are not modified.

---

### CMP

Compares two registers.

```assembly
CMP R0, R1
```

Result:

- Equal → `ZF = 1`, `CF = 0`
- First operand greater → `ZF = 0`, `CF = 0`
- First operand lower → `ZF = 0`, `CF = 1`

---

### JMP

Performs an unconditional jump.

```assembly
JMP address
```

---

### Conditional Jumps

| Instruction | Condition |
|-------------|------------|
| JZ | `ZF == 1` |
| JNZ | `ZF == 0` |
| JC | `CF == 1` |
| JNC | `CF == 0` |
| JE | `ZF == 1` |
| JL | `CF == 1` |
| JG | `CF == 0 && ZF == 0` |

All jumps use 16-bit addresses.

---

### PUSH

Pushes a register value onto the stack.

```assembly
PUSH R0
```

The stack grows downward.

---

### POP

Pops a value from the stack into a register.

```assembly
POP R1
```

---

### STOP

Stops execution.

```assembly
STOP
```

---

## Stack Protection

The VM detects:

- Stack overflow
- Stack underflow

Execution stops if one of these conditions occurs.

---

## Example Program

```assembly
LOAD R0, 10
LOAD R1, 20

ADD R0, R1

CMP R0, R1
JG greater

STOP

greater:
LOAD R2, 1
STOP
```

---

## Implementation Details

- Language: C
- Compiler: Python
- Register size: 8 bits
- Address size: 16 bits
- Endianness: Big-endian instruction addresses

Jump addresses are encoded as:

```text
high_byte low_byte
```

and reconstructed as:

```text
address = low_byte | (high_byte << 8)
```
